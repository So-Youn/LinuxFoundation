# Software Package

>  설치하고 나면, 모든 정보들은 version 상태 및 업데이트 정보를 편하게 조회할 수 있는 내부 DB에 로컬로 저장된다.

* 패키지 유형
  * Binary packages : deployment를 위해 준비된 files
  * source packages : binary packages를 생성하는데 사용되며, 항상 source package로부터 binary package를 rebuild할 수 있다. 
  * architecture-independent
  * Meta-packages

* package management system 종류
  * RPM(Red Hat Package Manager) : 레드헷이 개발한 패키지 관리 유틸리티
  * dpkg(Debian package)
* packaging Tool Levels and Varieties
  * Low Level Utilities 
  * High Level Utilities : 의존적이던 문제 해결





## RPM

---

1. Linux System에서 설치, 검증, 업그레이드,제거 등을 control하는데 사용된다.
2. 문제를 일으킬 수 있는 operation*(다른 패키지가 종속된 패키지를 제거하거나 시스템에 다른 소프트웨어를 먼저 설치해야 할 때 패키지 설치)*은 completion전에 차단된다.
3. 특정 파일의 원본, 설치된 버전, 제대로 설치되었는지의 여부 등을 쉽게 확인할 수 있다.
4. 디스크 공간을 확보하기 위해 전체 패키지를 쉽게 제거할 수도 있다.

---

**Package File Names**

* RPM standard에 따라 특정한 정보를 표현할 수 있도록 이름이 명명된다.

```markdown
# binary RPM package
<name>-<version>-<release>.<distro>.<architecture>.rpm
ex. sed-4.5-2.e18.x86_64.rpm

# source RPM package
<name>-<version>-<release>.<distro>.src.rpm
ex. sed-4.5-2.e18.src.rpm

* <distro>는 종종 패키지의 출처(repository)를 지정
```
