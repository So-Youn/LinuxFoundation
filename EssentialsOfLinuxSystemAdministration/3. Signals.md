## Signal

* Signal은 `IPC(Inter-Process Communication)` 및 비동기 event에 대한 process를 알린다.
  * interrupt의 일종
  * exceotion, programming error
  * `kill` : 프로세스에 신호를 보내기 위해 만들어진 명령어

- **SIGKILL** (#9)
- **SIGSTOP** (#19)

* 시그널 종류

  * `SIGTERM [default signal (#15)]` : death를 예방하기 위함, 가능한 정상 종료 시키는 signal

  * `SIGKILL(#9)` : 프로세스 강제 종료

    ```shell
    $ kill -9 1234
    
    $ kill -SIGTERM 1234
    ```

  * `SIGHUP` : 데몬 관련 환경 설정 파일을 변경시키고, 변경된 내용을 적용하기 위해 재시작할 때 `SIGHUP` 사용

* signal에 대한 모든 정보 열람

  * ```shell
    $ man 7 signal
    ```

## killall / pkill

> `killall`은 user가 충분한 권한을 가졌다고 가정했을 때, 모든 프로세스를 죽일 수 있다.
>
> * process ID보다는 command name을 사용한다

```shell
$ killall bash
$ killall -9 bash
$ killall -SIGKILL bash
```

> `pkill`은 selection criteria를 이용해서 process에 신호를 보낸다.
>
> `pkill`은 kill과 유사하지만, pid 대신에 이름을 사용한다.
>
> * `-P`: ppid
> * `-G`: 기드
> * `-U` : uid

```shell
# rsyslog가 구성파일을 다시 읽게 한다.
$ pkill -HUP rsyslogd
```
