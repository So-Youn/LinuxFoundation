>  rpm과 dpkg와 같은 lower-level package utilitie들은 특정 sw packages를 설치하고 이미 설치된 sw들을 다룬다
>
> Def, yum, apt, zypper와 같은 higher-level package management system들은 그런 많은 sw들을 통합한다.
>
> RHEL/CentOS 7에서 8로 전환하는 동안 dnf는 yum을 대체했고, 페도라는 yum을 훨씬 더 오래 사용하고 있다.

# dnf

* repository를 위한 config 파일들은 `/etc/yum.repos.d` 에 있으며, `.repo` 확장자를 가진다.

  ```shell
  [root@eef676fd512f yum.repos.d]# ls /etc/yum.repos.d/
  CentOS-Linux-AppStream.repo  CentOS-Linux-ContinuousRelease.repo  CentOS-Linux-Devel.repo   CentOS-Linux-FastTrack.repo		CentOS-Linux-Media.repo  CentOS-Linux-PowerTools.repo
  CentOS-Linux-BaseOS.repo     CentOS-Linux-Debuginfo.repo	  CentOS-Linux-Extras.repo  CentOS-Linux-HighAvailability.repo	CentOS-Linux-Plus.repo	 CentOS-Linux-Sources.repo
  ```

  * dnf는 성능 향상을 위해 정보와 데이터베이스를 캐싱하며, 캐싱된 정보를 제거하기 위해서는 다음과 같은 command를 실행시킨다.

  ```shell
  $ dnf clean [ packages | metadata | expire-cache | rpmdb | plugins | all ]
  ```

* dnf에 대한 자세한 내용 : 

  [Link]: https://docs.fedoraproject.org/en-US/quick-docs/dnf/

* dnf Queries

  ```markdown
  Searches for package with keyword in it:
  
  $ dnf search keyword
  
  Displays information about a package:
  
  $ dnf info package-name
  
  Lists packages installed, available, or updates:
  
  $ dnf list [installed | updates | available ]
  
  Shows information about package groups installed, available and updates:
  
  $ dnf grouplist
  
  Shows information about a package group:
  
  $ dnf groupinfo packagegroup
  
  Shows the owner of the package for file:
  
  $ dnf provides /path/to/file
  
  Please note the need to use at least one / in the file name, which can be confusing.
  ```

* update될 내용이 있는지 확인하기

  ```shell
  [root@eef676fd512f yum.repos.d]# dnf update

  Dependencies resolved.
  Nothing to do.
  Complete!
  [root@eef676fd512f yum.repos.d]# dnf check-update
  ```
